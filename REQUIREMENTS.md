# 项目需求说明

## 目标

开发一个本地网页版中文文字动效序列帧生成器，用于生成可导入 AE、Premiere、DaVinci Resolve 等软件的 PNG 序列帧。

第一阶段只验证一件事：代码生成的中文文字动效，是否能达到可用于商业视频合成的质感。

## 当前 MVP 范围

- Vite + TypeScript。
- Canvas 2D 渲染。
- 中文优先的文字输入和字体配置。
- 实时预览。
- 播放、暂停、拖动进度。
- PNG 序列帧 ZIP 导出。
- 效果模板选择。
- 支持其他模型生成符合接口的效果模板代码。

## 视觉原则

- 文字效果优先。
- 全屏装饰默认关闭。
- 不默认生成大边框、全屏 HUD、复杂背景 UI。
- 可以保留围绕文字的局部小线条、小方块、小框线，它们应服务文字表现。
- 中文文本必须是第一优先使用场景。

## 不做的事

- 不做视频导出。
- 不接 FFmpeg。
- 不做 AI prompt 生成动画。
- 不做复杂节点系统。
- 不做 Electron / Tauri 打包。
- 不做音频。
- 不做模板商店。
- 不做用户账户。
- 不做云端保存。
- 不做复杂多轨 timeline。
- 不做完整字幕软件。

## 模板接入目标

其他模型，例如 Gemini，可以生成一个符合项目接口的 TypeScript 效果文件。用户将该文件放入 `src/effects/templates/` 后，页面可以自动识别并显示这个效果。

模板代码只负责绘制当前帧，不负责页面 UI、导出、文件下载、DOM 操作或网络请求。模板可以使用 `decoration` 控制文字周边局部装饰，但不应默认生成全屏边框。

## 字体需求

- 支持常用中文字体预设。
- 支持用户手动输入 `font-family`。
- 支持浏览器允许时扫描本机字体。
- 如果浏览器不支持系统字体扫描，页面必须继续可用。

## 后续方向

- 增加更多高质量文字效果模板。
- 优化中文字体排版和字重控制。
- 根据需要引入 PixiJS/WebGL 提升性能。
- 后续可以借鉴确定性视频渲染管线，但当前阶段不接视频导出。
